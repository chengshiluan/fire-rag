# 消防知识库（RAG）数据调研与爬取需求文档
> 原始 PRD | 版本 v0.1 | 草拟日期：2026-06-12

## 一、项目背景与目标

构建面向消防领域的检索增强生成（RAG）知识库，为消防设计审查、合规咨询、应急处置、培训问答等场景提供权威、准确、可追溯的知识检索能力。

本阶段目标：明确数据范围、确定数据源、完成可行性调研，并产出一套可执行的数据采集（爬取/下载/解析）方案，为后续工程开发提供依据。

## 二、知识库覆盖范围（数据分类）

按权威层级与内容类型，将消防知识库数据划分为以下五类：

| 类别 | 内容说明 | 数据规模预估 |
|------|----------|-------------|
| 法律/行政法规 | 《消防法》及配套行政法规、部门规章 | 数十部 |
| 国家标准（GB） | 建筑设计防火规范、自动喷水灭火系统设计规范、火灾自动报警系统设计规范等核心规范 | 约20-30部 |
| 地方性法规/标准 | 各省市消防条例、地方设计导则 | 约30-50部 |
| 行业规程/技术指南 | 消防验收规程、应急处置技术规程等 | 数十至上百份 |
| 补充资料 | 事故案例、FAQ问答对、政策解读 | 数百至数千条（可后期补充） |

核心结论：优先保证前四类（法规+标准）的完整性与准确性，预计折算纯文本规模在数千万字符级别，切分后约1-5万个检索chunk，足以支撑初版RAG。

## 三、数据源调研清单

### 3.1 法律法规类 —— 国家法律法规数据库

| 项目 | 内容 | 备注 |
|------|------|------|
| 网址 | flk.npc.gov.cn | 全国人大官方法规库 |
| 数据形式 | 网页HTML正文 | 结构清晰，可直接解析为文本，质量最高 |
| 获取方式 | 站内检索接口（关键词"消防"），批量拉取结果列表→逐条抓取正文 | 需调研其检索接口的请求参数与返回格式 |
| 调研任务 | 1) 确认检索接口URL与参数格式 2) 确认是否需要登录/有无频率限制 3) 抓取样例并验证正文结构 | |

### 3.2 国家标准类（GB系列）—— 国家标准全文公开系统

| 项目 | 内容 | 备注 |
|------|------|------|
| 网址 | openstd.samr.gov.cn | 国家市场监督管理总局 |
| 数据形式 | 在线预览（分页图片+水印） | 无法直接复制文本，是本项目最大难点 |
| 获取方式 | 逐页截图 → OCR识别 → 文本校对 | 工作量大，需评估OCR准确率（涉及大量表格、图例） |
| 调研任务 | 1) 抽样3-5部核心规范（如GB50016）试做截图+OCR流程 2) 评估OCR文本质量与人工校对成本 3) 调研是否存在正版电子版采购渠道（更省成本、更准确） | |
| 替代方案 | 评估购买正版电子规范（设计院常用做法），可大幅降低数据处理成本和版权风险 | |

### 3.3 部委规章/政策文件 —— 应急管理部官网

| 项目 | 内容 | 备注 |
|------|------|------|
| 网址 | www.mem.gov.cn | 政策法规栏目 |
| 数据形式 | PDF/HTML公告，可直接下载 | 质量较好，部分为扫描件PDF |
| 获取方式 | 栏目列表分页抓取 → 下载PDF/解析HTML → pdfplumber提取文本 | |
| 调研任务 | 1) 梳理政策法规栏目结构与URL规律 2) 抽样下载验证PDF是否为文本型或扫描型 | |

### 3.4 地方性法规/标准

| 项目 | 内容 | 备注 |
|------|------|------|
| 来源 | 各省人大官网 / 地方应急管理厅官网 | 网站结构差异大，需逐一调研 |
| 调研任务 | 1) 先确定优先覆盖的省市清单（如业务覆盖城市） 2) 逐一调研目标网站的法规检索入口与数据形式 | 建议先做3-5个重点城市试点 |

### 3.5 可参考的现有开源资源

- LawRefBook/Laws（GitHub）：通用法律法规库，可直接复用其中已收录的《消防法》文本，并参考其 request.py 抓取国家法律法规数据库的实现
- ImCa0/just-laws（GitHub）：法律文本VuePress文库，消防相关内容有限，但其"Skill驱动批量收录+进度跟踪"的自动化流程思路可借鉴，用于设计消防法规收录流程

## 五、技术方案要点（供调研参考）

### 5.1 采集流程
- 文本类来源（flk.npc.gov.cn、应急管理部HTML公告）：requests + 解析库（BeautifulSoup/lxml）批量抓取正文
- PDF类来源：批量下载 → pdfplumber/camelot 提取文本与表格
- 图片/在线预览类来源（国标GB）：截图 → OCR（如PaddleOCR）→ 人工校对

### 5.2 数据处理与入库
- 文本清洗：去除页眉页脚、目录编号、水印干扰文字
- 结构化切分：按"条/款/项"或标准章节切分为chunk，保留法规名称、条款号等元数据用于引用追溯
- 元数据字段建议：法规名称、发布机关、发布/生效日期、层级（法律/国标/地方标准）、章节/条款号、来源URL

### 5.3 合规与版权注意事项
- 官方公开发布的法律法规文本本身不受版权限制，可正常采集使用
- 国家标准（GB）文本版权归属国家标准化机构，大规模复制需注意版权风险，建议优先核实是否需要采购授权
- 抓取频率需控制，避免对官方网站造成压力，遵守各网站robots协议
