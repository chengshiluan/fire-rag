# GB 消防标准获取方案调研报告

> 调研日期：2026-06-12
> 调研对象：国家标准全文公开系统 (openstd.samr.gov.cn) 及正版采购渠道
> 调研人：数据采集工程师

---

## 一、核心发现（Executive Summary）

### 关键结论

**GB 50016《建筑设计防火规范》、GB 50084《自动喷水灭火系统设计规范》、GB 50116《火灾自动报警系统设计规范》均为"工程建设"类国家标准，不在 openstd.samr.gov.cn 收录范围内。** 该系统首页明确声明："食品安全、环境保护、工程建设方面的国家标准未纳入本系统"。经验证，在 openstd 搜索 GB 50016 返回 0 条结果，确认了这一事实。

因此，**截图+OCR 方案对这三部核心消防规范不可行**（技术上不可行，因为根本没有在线预览页面可供截图）。

### 推荐方案

| 优先级 | 方案 | 预估成本 | 说明 |
|--------|------|---------|------|
| **推荐** | 正版电子版采购 | 约 2000-4000 元 | 通过 spc.org.cn / ndls.org.cn 购买，合法、可检索、可复制文本 |
| 备选 | 纸质版购买 + 专业扫描 | 约 3000-5000 元 | 如无电子版在售则采购纸质版后委托专业扫描 |
| 不推荐 | 截图 + OCR | N/A | 目标规范不在 openstd 收录范围，无法进行 |

### 2.2 排除范围验证

在 openstd 搜索 "GB 50016" 返回 0 条结果，搜索页面提示："可能的原因如下：1.您所查询的标准不在公开范围之内 2.您的查询条件输入有误"。确认消防工程类规范不在 openstd 系统中。

### 2.3 在线预览机制技术分析

尽管目标消防规范不在 openstd 中，以下仍对其技术机制进行分析（供其他标准参考）。

#### 2.3.1 历史演进

| 时间 | 技术方式 | 说明 |
|------|---------|------|
| 2021年前 | 明文 PDF (手机版) | gb688.cn 手机版直接提供明文 PDF |
| 2021年 | pdf.js + AES-128-CBC 加密 | 过渡到加密 PDF，分段传输，浏览器端解密 |
| **2022年4月至今** | **图片拼接模式 (Canvas/Image)** | 弃用 pdf.js，全面使用图片渲染方式 |

#### 2.3.2 当前机制（2022年4月后）

1. 在线预览页面将每页标准内容渲染为 Canvas 或 Image 元素
2. 页面 DOM 中包含多个 `canvas[id^=canvas_]` 或 `<img>` 元素
3. 支持分页浏览，用户滚动时动态加载后续页面
4. 可通过浏览器截图或打印功能获取图片
5. **直接下载功能**：部分标准提供"下载标准"按钮，但下载的是 FileOpen DRM 加密的 PDF，需安装 FileOpen 插件 + Adobe Reader 才能打开

#### 2.3.3 旧版加密机制（2021-2022，已过时）

```
旧版流程：
1. 页面包含加密变量 var HCNO="base64密文"
2. pdf-work.wasm 解密 HCNO → 获得 key:iv:hcno
3. 使用 AES-128-CBC 解密服务端返回的分段加密数据
4. 解密后的 PDF 数据由 pdf.js 渲染

API 端点：
- 预览页面: http://c.gb688.cn/bzgk/gb/showGb?type=online&hcno={HCNO}
- 加密文件: http://c.gb688.cn/bzgk/gb/viewGb?type=online&hcno={HCNO}
- 下载页面: http://c.gb688.cn/bzgk/gb/showGb?type=download&hcno={HCNO}
```

### 2.5 OCR 方案可行性评估

#### 2.5.1 引擎对比

| 维度 | PaddleOCR | Tesseract 5 | EasyOCR |
|------|-----------|-------------|---------|
| 中文印刷体识别率 | **~95%** (PP-OCRv5) | ~85-90% | ~88-92% |
| 表格识别 | **内置 PP-Structure** | 不支持 | 不支持 |
| 布局分析 | **内置** | 基础 | 不支持 |
| GPU 加速 | 支持 | 不支持(CPU only) | 支持 |
| 手写识别 | ~73% | ~45% | ~62% |
| 许可证 | Apache 2.0 | Apache 2.0 | Apache 2.0 |
| 模型大小 | ~500MB | ~10MB | ~500MB |

**推荐方案**：PaddleOCR (PP-OCRv5 + PP-Structure)，提供最高中文识别率和内置表格/布局分析能力。

#### 2.5.2 表格和图例识别难度评估

- **表格识别难度：高**。消防规范中包含大量结构复杂的表格（如耐火等级表、防火间距表、喷头布置参数表等）。
  - PaddleOCR 的 PP-Structure 可输出 HTML 格式的表格结构，准确率约 85-90%
  - 复杂合并单元格、跨页表格需要人工校对
- **图例识别难度：极高**。消防规范中包含大量建筑平面图、系统示意图、管道布置图。
  - OCR 引擎无法理解图示语义
  - 图例仅能保留截图，无法转为结构化文本
  - 图例中的数据标注（数字、箭头、符号）可能需要人工解读
- **公式识别难度：中高**。部分规范包含计算公式（如喷头流量计算、水力计算等）。
  - PaddleOCR-VL 支持公式识别，但准确率待验证

#### 2.5.3 人力成本估算（假设规范在 openstd 中）

假设 OCR 处理 20 部规范：

| 项目 | 估算值 |
|------|--------|
| 每部规范平均页数 | ~150 页（含条文说明） |
| 20 部总页数 | ~3,000 页 |
| OCR 自动识别时间 | ~2秒/页 = 1.7 小时（GPU）/ ~8 小时（CPU） |
| 每页校对时间（纯文字） | ~1 分钟 |
| 每页校对时间（含表格） | ~3 分钟 |
| 表格页占比 | ~30% |
| 总校对时间 | (3,000 * 70% * 1) + (3,000 * 30% * 3) = 2,100 + 2,700 = 4,800 分钟 = 80 小时 |
| 图例页处理时间（标注+描述） | 约 300 页 * 5 分钟 = 25 小时 |
| **总人力工时** | **约 105 小时 ≈ 13-15 人天** |

#### 2.5.4 OCR 方案总成本估算

| 成本项 | 金额 |
|--------|------|
| 人工校对（15 人天 * 800 元/天） | 12,000 元 |
| GPU 服务器（按需，可忽略） | ~500 元 |
| 质量审核（3 人天 * 1,000 元/天） | 3,000 元 |
| **OCR 方案总成本** | **约 15,000-16,000 元** |

### 3.4 电子版格式详细说明

#### FileOpen 加密 PDF（中国标准出版社/spc.org.cn）
- **格式**：PDF + FileOpen DRM 加密
- **阅读要求**：Adobe Reader + FileOpen 插件
- **文本复制**：受限，取决于出版社设置
- **打印**：通常允许有限次打印
- **设备限制**：通常绑定 3 台设备
- **优点**：正版授权，内容权威，无水印
- **缺点**：使用不便，移动端支持差，不适合程序化处理

#### True-PDF（第三方英文版渠道）
- **格式**：无 DRM 的标准 PDF
- **文本复制**：完全可复制
- **优点**：使用自由，可程序化处理
- **缺点**：仅有英文翻译版，单价高（$180+/部）

**推荐方案 A**：正版电子版直接采购，成本最低、质量最高、法律风险为零。

若需要可全文检索的文本数据用于构建 RAG 知识库，建议：
1. 优先采购电子版（PDF）

3. 对表格和公式进行结构化标注
4. 保留原始 PDF 作为权威依据

## 四、最终推荐方案

### 推荐方案：正版采购 + PDF 文本提取

```
第1步：预算确认与清单制定
  - 确定需采购的消防规范清单（20-30部）
  - 预算估算：约 ¥2,000 (电子版) 或 ¥3,000 (纸质版)

第2步：正版电子版采购
  - 优先渠道：spc.org.cn（中国标准在线服务网）
  - 备选渠道：ndls.org.cn（国家数字标准馆）
  - 联系客服确认批量采购折扣：400-168-0010

第3步：文本提取与结构化
  - 使用 PyMuPDF (fitz) 提取 PDF 文本
  - 使用 pdfplumber 提取表格数据
  - 建立章节结构索引
  - 构建 RAG 知识库文档索引

第4步（可选）：OCR 补充
  - 仅当电子版为扫描件且文字不可复制时
  - 使用 PaddleOCR 进行 OCR
  - 人力投入约 13-15 人天用于校对
```
