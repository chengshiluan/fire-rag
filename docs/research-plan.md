# 消防知识库（RAG）数据调研计划

> 版本 v0.1 | 日期 2026-06-12 | 基于 PRD v0.1

---

## 一、需求理解与复述

### 项目目标
构建面向消防领域的 RAG 知识库，服务四个场景：**消防设计审查、合规咨询、应急处置、培训问答**。核心要求是权威、准确、可追溯。

### 数据范围（五类，按优先级排序）

| 优先级 | 类别 | 规模 | 技术难度 |
|--------|------|------|----------|
| P0 | 法律/行政法规 | 数十部 | 低 — HTML正文，直接解析 |
| P0 | 国家标准（GB） | 20-30部 | **极高** — 在线预览为分页图片+水印，需OCR |
| P1 | 地方性法规/标准 | 30-50部 | 中高 — 网站结构各异 |
| P1 | 行业规程/技术指南 | 数十至上百份 | 中 — PDF为主 |
| P2 | 补充资料 | 数百至数千条 | 低 — 可后期补充 |

### 文本规模估算
- 四类核心数据预计数千万字符
- 切分后约 10,000-50,000 个检索 chunk
- 足以支撑初版 RAG 系统

## 二、数据源分析与技术难点预判

### 2.1 各数据源特征矩阵

```
数据源                   格式      反爬难度   是否需要登录   采集方式
─────────────────────────────────────────────────────────────────
flk.npc.gov.cn          HTML      低         不需要         API检索 + 正文解析
openstd.samr.gov.cn     图片+水印  低         不需要         截图→OCR→校对
mem.gov.cn               PDF/HTML  低         不需要         下载→pdfplumber
地方人大/应急厅官网      各异      各异       各异           逐个定制脚本
```

### 2.2 最大难点：国家标准（GB）采集

openstd.samr.gov.cn 上 GB 规范为**分页图片+水印在线预览**，无法直接复制文本。这意味着：

1. **OCR 准确率是关键瓶颈**：消防规范含大量表格（耐火等级表、防火间距表）、图例（疏散示意图、管道布置图）、化学分子式、特殊符号
2. **人工校对工作量大**：按每部规范 100-300 页，20-30 部 = 2000-9000 页需 OCR + 校对
3. **版权风险**：GB 文本版权归国家标准化机构

**建议优先调研正版电子版采购渠道**，若采购成本可接受（通常几千到几万元），可避免大量 OCR 与校对工作。

## 三、调研阶段划分

### Phase 1：可行性验证（预计 2-3 天）

本阶段只采样、不批量。目标是快速验证每个数据源是否可采集、采集质量是否达标。

#### Task 1.1：flk.npc.gov.cn 检索接口调研
- **目标**：确认"消防"关键词检索的 API 接口
- **方法**：
  1. 浏览器打开 flk.npc.gov.cn，使用站内搜索"消防"，抓取 Network 请求
  2. 分析请求 URL、参数（关键词、分页、排序）、返回格式（JSON/HTML）
  3. 测试是否有频率限制、是否需要 token/cookie
  4. 抓取 3-5 条结果详情页，确认正文 HTML 结构
- **产出**：API 接口文档 + 样例抓取脚本 + 5 条样例正文
- **预计工时**：4h

#### Task 1.2：openstd.samr.gov.cn 截图+OCR 流程验证
- **目标**：评估 GB 标准 OCR 方案的可行性与成本
- **方法**：
  1. 选取 3 部核心规范抽样（建议 GB 50016《建筑设计防火规范》、GB 50084《自动喷水灭火系统设计规范》、GB 50116《火灾自动报警系统设计规范》）
  2. 每部取 5-10 页（含纯文字页、含表格页、含图例页），逐页截图
  3. 使用 PaddleOCR 或 Tesseract 进行 OCR 识别
  4. 对比原文评估准确率，特别关注表格、数字、特殊符号
  5. 估算单页校对时间 × 总页数 = 人工校对总工时
- **产出**：OCR 准确率评估报告（分文字/表格/图例三类）+ 工时估算
- **预计工时**：8h

#### Task 1.3：正版电子版规范采购渠道调研
- **目标**：对比自建 OCR vs 采购正版电子版
- **方法**：
  1. 调研中国标准出版社、标准在线等平台的电子版售价
  2. 确认电子版格式（PDF文本型？还是扫描件？是否可复制文本？）
  3. 确认授权范围（是否允许用于知识库构建？）
  4. 计算 ROI：采购成本 vs OCR+校对人力成本
- **产出**：采购 vs OCR 成本对比表 + 推荐方案
- **预计工时**：3h

#### Task 1.4：应急管理部官网结构调研
- **目标**：梳理 mem.gov.cn 政策法规栏目，验证 PDF 提取效果
- **方法**：
  1. 分析政策法规栏目 URL 规律与分页机制
  2. 抽样下载 5-10 份消防相关文件
  3. 对 PDF 分类：文本型（直接提取）vs 扫描型（需 OCR）
  4. 用 pdfplumber 提取文本型 PDF，评估提取质量
- **产出**：栏目结构文档 + PDF 分类统计 + 样例提取文本
- **预计工时**：4h

#### Task 1.5：地方性法规试点调研
- **目标**：完成 1-2 个重点城市的法规采集可行性验证
- **方法**：
  1. 先确定优先覆盖省市清单（建议：北京、上海、广东、浙江、四川）
  2. 选取 **北京市** 和 **上海市** 作为试点
  3. 调研两地人大官网 / 应急管理局官网的法规检索入口
  4. 采样 3-5 部地方消防法规，确认数据格式
- **产出**：地方试点调研报告 + 省市优先级清单
- **预计工时**：5h

### Phase 2：工程方案设计（预计 1-2 天）

基于 Phase 1 的验证结论，设计完整的工程方案。

#### Task 2.1：反爬策略评估与合规方案
- 各数据源 robots.txt 检查
- 频率限制测试与建议抓取间隔
- 是否需要代理 IP 池
- 合规声明文档

#### Task 2.2：数据采集架构设计
- 确定各数据源的技术栈（requests/playwright/selenium）
- 设计采集流水线：抓取 → 清洗 → 结构化 → 入库
- 设计增量更新机制（法规更新时如何同步）

#### Task 2.3：文本处理与 Chunk 策略设计
- 确定分块粒度（按"条/款/项"还是固定 token 数）
- 设计元数据 schema
- 设计重复/冲突检测策略

#### Task 2.4：产出《消防法规/标准采集清单》
- 完整的法规/标准名称列表
- 每个条目的：来源 URL、数据形式、获取方式、优先级、预计文本量

---

### Phase 3：原型脚本开发（与 Phase 2 并行，预计 2-3 天）

#### Task 3.1：flk.npc.gov.cn 批量采集脚本
- 关键词"消防"检索 → 获取全部结果列表
- 逐条抓取正文
- 文本清洗 + 格式化输出

#### Task 3.2：应急管理部 PDF 批量采集脚本
- 分页遍历政策法规栏目
- 批量下载 PDF/HTML
- pdfplumber 提取文本

#### Task 3.3：GB 标准截图+OCR 原型脚本（视 Phase 1.2 结论决定是否推进）

## 五、风险清单

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| GB OCR 准确率不达标（<85%） | 高 | 严重 | 优先推进正版采购调研 |
| 地方网站有验证码/反爬 | 中 | 中等 | 先试点验证，逐个攻克 |
| 某些网站需登录才能检索 | 中 | 中等 | 评估注册可行性 |
| PDF 全部为扫描件 | 中 | 中等 | pdfplumber 退化为 OCR 流程 |
| 数据量超出预期（>5万 chunk） | 低 | 中等 | 向量数据库选型时预留扩展空间 |
