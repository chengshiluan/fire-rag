## Part A：省市优先级评估

### 评估方法

基于四个维度进行综合评估（每项满分 5 分）：

| 维度 | 权重 | 说明 |
|------|------|------|
| 经济体量 | 30% | GDP 排名决定地区经济体量，与消防投入正相关 |
| 城市建设活跃度 | 25% | 城市更新、基建活跃度反映消防法规更新频率 |
| 消防法规完善度 | 25% | 地方消防法规数量、修订频率 |
| 数据可得性 | 20% | 是否有公开法规检索系统、是否支持数字化采集 |

### C.1 LawRefBook/Laws（GitHub）

**基本信息：**

| 属性 | 内容 |
|------|------|
| GitHub Repo | `https://github.com/LawRefBook/Laws` |
| 语言 | Python (91.9%), Shell (8.1%) |
| 许可证 | 未明确声明 |
| 数据来源 | 国家法律法规数据库 (flk.npc.gov.cn) |
| 输出格式 | Markdown + SQLite |

**request.py 核心逻辑分析：**

1. **API 交互**：通过 `RequestManager` 类封装对国家法律法规数据库 API 的请求
   - `getLawList(i)`: 分页获取法律列表（支持多页遍历，第1到60页）
   - `get_law_detail(id)`: 根据法律ID获取详情（标题、层级、正文文件列表）
   - 支持按类别参数过滤（如 `fgxlwj`=行政法规, `zdjg`=指定地区）

2. **地区过滤机制**（可直接复用）：
   ```python

   ("zdjg", "4028814858b9b8e50158bec45e9a002d&4028814858b9b8e50158bec500350031"), #上海
   ```

## 总体评估与建议

### 数据采集策略

**优先级 P0（立即开始）：**
- 北京、上海、广东

**优先级 P1（第二阶段）：**
- 江苏、浙江、山东

**优先级 P2（第三阶段）：**
- 四川、湖北

### 技术路线建议

1. **国家法律法规数据库 (flk.npc.gov.cn) API** 作为第一数据源（参考 LawRefBook/Laws request.py 的 `zdjg` 参数体系）
2. **各省市政府网站 HTML 采集** 作为补充（北京 beijing.gov.cn、上海 service.shanghai.gov.cn、广东 gd.gov.cn）
3. **多策略兜底**（参考 Law-Crawler-RPA-RAG-MCP 的 CrawlerManager 设计）
4. **结构化输出**：JSON（元数据 + 正文） + Markdown（阅读友好）双格式

### 关键风险

1. **网站改版**：政府网站可能调整页面结构，CSS 选择器需定期维护
2. **PDF 文档**：部分法规仅提供 PDF 版本，需集成 PDF 解析（pdfplumber / PyMuPDF）
3. **附件外部链接**：部分政府网站法规链接指向外部（如 moj.gov.cn），需处理跨站采集
4. **增量更新**：法规修订频繁（如北京消防条例2025年第三次修订），需建立版本追踪机制
